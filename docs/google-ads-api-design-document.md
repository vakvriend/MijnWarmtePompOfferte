# Google Ads API Design Document

## Project

**Application name:** Vakvriend Ads Builder  
**Company / advertiser:** Vakvriend  
**Primary domain:** https://www.vakvriend.nl  
**Campaign domains:** warmtepomp campaign domains, including warmtepompdenhaag.nl and warmtepomphilversum.nl  
**Environment:** WordPress theme module inside the Vakvriend warmtepomp campaign website  
**Requested Google Ads API use:** Campaign creation/management and reporting  

## Purpose

Vakvriend Ads Builder is an internal tool used by Vakvriend to create, review, launch and optimize Google Search campaigns for its own lead-generation websites. The tool is not offered to third-party advertisers and is not a public SaaS product.

The business goal is to create controlled local Search campaigns for warmtepomp advies, woningcheck, ISDE-subsidie and related installation intent searches. The system generates campaign structures per domain/city and connects Google Ads performance data to first-party lead data from WordPress.

## Users

The tool is used only by authorized Vakvriend administrators inside WordPress.

User roles:

- **Administrator:** can configure API credentials, generate campaigns, export CSV files, create paused campaigns through the API and view reporting.
- **Campaign operator:** can review generated plans and performance results.

No end user or website visitor can access the Ads Builder.

## Google Ads API Features Used

The tool uses the Google Ads API for:

- Listing accessible customers for connection testing.
- Creating Search campaign resources in a paused state.
- Creating campaign budgets.
- Creating ad groups.
- Creating keyword criteria.
- Creating responsive search ads.
- Creating campaign-level negative keywords.
- Fetching campaign, ad, keyword and performance metrics.
- Connecting Google Ads performance data to local WordPress lead data through UTM parameters.

Relevant Google Ads API concepts:

- Google Ads accounts use a hierarchy of customer, campaign, ad group and ad/ad criterion resources.
- Reporting is performed with `GoogleAdsService.SearchStream`.
- Mutations are performed with mutate operations.

References:

- Google Ads API access levels and permissible use: https://developers.google.com/google-ads/api/docs/api-policy/access-levels
- Google Ads API structure: https://developers.google.com/google-ads/api/docs/concepts/api-structure
- REST interface design: https://developers.google.com/google-ads/api/rest/design/overview

## Access Level Requested

The application requires production access because campaigns are created and managed for live Vakvriend Google Ads accounts.

Requested permissible use:

- **Ad creation / management**
- **Reporting**

Basic Access is sufficient at the current expected volume. The tool is designed for a limited number of internal campaigns and local landing pages, not high-volume agency-scale account management.

## Authentication and Authorization

Authentication uses OAuth 2.0 with a refresh token generated by an authorized Google Ads user.

Stored settings:

- Developer token
- OAuth client ID
- OAuth client secret
- Refresh token
- Customer ID
- Optional login customer ID for manager-account access
- API version

Credentials are stored in WordPress options and only visible/editable to WordPress administrators with `manage_options`.

The API follows Google Ads account permissions. If the OAuth user does not have access to a customer account, the tool cannot access or mutate that account.

## Data Flow

### Campaign Creation Flow

1. Administrator opens **WordPress Admin > Ads Builder**.
2. Administrator selects:
   - domain
   - city
   - daily budget
   - focus, such as advice, subsidy, hybrid, Qvantum/Nibe or warmtepompboiler
3. System generates a campaign plan:
   - campaign name
   - ad groups
   - keywords
   - responsive search ad copy
   - A/B ad variants
   - negative keywords
   - UTM-tagged final URLs
4. Administrator reviews the campaign plan.
5. Administrator can export CSV for Google Ads Editor or create the campaign through the API.
6. If created through the API, campaign, ad groups and ads are created with `PAUSED` status.
7. Administrator reviews and manually enables the campaign in Google Ads after checking settings such as location targeting, budget and compliance.

### Reporting Flow

1. Administrator opens **Ads Builder > Resultaten & A/B**.
2. Administrator clicks **Google Ads resultaten ophalen**.
3. Tool requests Google Ads performance data using `GoogleAdsService.SearchStream`.
4. Results are stored locally in WordPress for dashboard display.
5. Tool combines Google Ads data with WordPress lead data by matching:
   - `utm_campaign`
   - `utm_content`
   - landing domain
6. Dashboard shows:
   - impressions
   - clicks
   - CTR
   - cost
   - Google Ads conversions
   - WordPress leads
   - average lead score
   - phone/postcode completion

## A/B Testing Design

Each generated campaign contains two default ad messaging variants:

- **Variant A:** advice / woningcheck angle  
  UTM content: `a_advies_woningcheck`

- **Variant B:** subsidy / ISDE angle  
  UTM content: `b_subsidie_isde`

Each variant receives its own responsive search ad text and final URL with `utm_content`.

The system evaluates variants on:

- impressions
- clicks
- CTR
- cost
- Google Ads conversions
- WordPress leads
- average local lead score
- whether the lead also supplied phone/postcode after submission

This avoids optimizing only for clicks. The goal is lead quality and lead volume.

## Data Stored

### Google Ads API Settings

Stored in WordPress options:

- developer token
- OAuth client ID
- OAuth client secret
- refresh token
- customer ID
- login customer ID
- default daily budget
- API version

### Generated Campaign Data

Campaign data is generated dynamically and can be exported to CSV. It includes:

- campaign name
- ad group names
- keywords
- negative keywords
- responsive search ad copy
- final URLs with UTM parameters
- daily budget
- paused status

### Reporting Data

Stored locally after API refresh:

- campaign name
- ad group name
- ad ID
- final URL
- impressions
- clicks
- cost
- conversions
- CTR
- keyword text
- keyword match type

### Lead Data

Lead data is stored in WordPress as private custom post type entries. It includes:

- name
- email
- optional phone
- optional postcode
- city
- domain
- selected system preference
- gas usage range
- lead score
- UTM source, medium, campaign, term and content
- Google Ads click identifiers when available

## Privacy and Personal Data

Google Ads reporting data is aggregated performance data and does not contain personally identifiable visitor data.

WordPress lead data is first-party data submitted by the visitor. It is used to:

- send the requested woningcheck
- contact the lead when needed
- evaluate campaign quality
- improve campaign performance

Personal data is not sent back to Google Ads through the current implementation.

No customer match, enhanced conversions or offline conversion upload is implemented in this phase.

## Security Controls

- Ads Builder is available only to logged-in WordPress administrators.
- All actions use WordPress capability checks.
- Admin form submissions use WordPress nonces.
- API campaigns are created in `PAUSED` status.
- Ads are created in `PAUSED` status.
- Ad groups are created in `PAUSED` status.
- Administrator review is required before campaigns can spend money.
- User input is sanitized before use.
- Google Ads customer IDs are stored without dashes and sanitized to digits.
- API errors are shown in the admin interface for troubleshooting.

## Budget and Spend Control

The system never creates active campaigns by default.

Controls:

- generated campaigns use an explicit daily budget
- default budget is configurable
- campaign creation is confirmed by the administrator
- campaign status is paused at creation
- manual review in Google Ads is required before serving

## API Operation Volume

Expected usage is low.

Typical daily use:

- 1-20 campaign plan generations in WordPress without API calls
- 0-5 campaign creation calls when launching new city campaigns
- 1-5 reporting refreshes per day

The current design is compatible with Basic Access operation limits.

## Error Handling

The tool handles API failures by:

- displaying the returned API error in WordPress admin
- not retrying destructive or mutating operations automatically
- leaving campaigns paused when creation succeeds
- storing the latest reporting result and fetch status

Partial failure is currently disabled for campaign creation to avoid incomplete campaign structures.

## Services and Endpoints

Current REST endpoints:

- `customers:listAccessibleCustomers` for API connection testing
- `customers/{customer_id}/googleAds:mutate` for campaign, budget, ad group, keyword and ad creation
- `customers/{customer_id}/googleAds:searchStream` for reporting

## Current Implementation Status

Implemented:

- WordPress Ads Builder admin page
- campaign generator per domain/city
- CSV export
- Google Ads API settings page
- API connection test
- paused campaign creation through API
- A/B ad variant generation
- UTM tagging per variant
- reporting fetch from Google Ads API
- local lead reporting by `utm_campaign` and `utm_content`

Not yet implemented:

- automatic bid optimization
- automatic campaign activation
- offline conversion upload
- enhanced conversions
- automated budget changes
- customer match
- third-party account management

These are intentionally excluded from the initial design to keep the system controlled and compliant.

## Compliance Position

The application is designed for internal first-party campaign management. It does not sell or expose Google Ads functionality to third parties.

The system is intentionally conservative:

- campaigns are created paused
- users manually review before activation
- reporting is used for optimization advice
- personal lead data remains in WordPress
- no automatic spend changes are made

## Reviewer Summary

Vakvriend Ads Builder is an internal WordPress-based Google Ads tool for creating and analyzing Search campaigns for Vakvriend’s own lead-generation websites. It uses the Google Ads API for campaign creation and reporting, with strict manual controls and paused-by-default campaign creation. The tool connects Google Ads performance to first-party lead outcomes using UTM parameters so campaign decisions are based on real lead quality rather than clicks alone.
